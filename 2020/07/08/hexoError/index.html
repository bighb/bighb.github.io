折腾Hexo的奇葩经历

在投简历之前，我决定搭建自己的网站，看到了hexo之后，我就决定使用它，在完成我的博客网站的过程中，遇到了奇奇怪怪的错误，对于一枚前端菜鸟来讲，没有浏览器console.log使用，使用hexo产生的错误，让我措手不及，接下来，我就讲述一下我的踩坑经历

Error-tags

相信大家在刚刚部署完hexo，在本地访问的时候，就已经迫不及待的点击，hexo给我们带来的各种按钮，我也一样，当点击标签时，出现Cannot not get /tags/ 我当时就觉得，我可能没有配置我的标签页面，因此，我还不犹豫的开始折腾tegs标签页面，按照官网的说法，我在source/目录下建立了tags目录，经历了一番编辑index.md文件之后，终于跟网上说的一样了

添加 type: “tags”
对于---要不要删掉，网上方法也有不同的意见，反正我是各尝试了一遍，好了，开始本地测试了hexo s，打开浏览器localhost:4000
还是出现错误，同样的错误，经历不同的尝试，终于发现了这个巨坑

解决Error-tags

因为我是武汉的因此在站点配置文件中，将timezone设置为：timezone: Asia/WuHan

最后改成timezone: Asia/Shanghai，终于出现了，一切正常运行着tags页面

Error-categories

>跟解决Error-tags一样

Hexo的一些基本命令

hexo g #完整命令为hexo generate,用于生成静态文件

hexo s #完整命令为hexo server,用于启动服务器，主要用来本地预览

hexo d #完整命令为hexo deploy,用于将本地文件发布到github等git仓库上

hexo n “my article” #完整命令为hexo new,用于新建一篇名为“my article”的文章
